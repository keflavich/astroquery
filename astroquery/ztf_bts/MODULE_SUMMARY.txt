ZTF Bright Transient Survey (BTS) Module for Astroquery
================================================================================

SUMMARY
=======

A complete astroquery module has been created to interface with the Zwicky
Transient Facility (ZTF) Bright Transient Survey Sample Explorer at:
https://sites.astro.caltech.edu/ztf/bts/explorer.php

The module provides a Pythonic interface for querying the ZTF BTS catalog using
various search criteria including object names, sky coordinates, classification
types, redshift ranges, temporal properties, and photometric characteristics.

QUICK START
===========

Installation:
------------
The module is ready to use within the astroquery package:

    from astroquery.ztf_bts import ZTFBTS

Basic Usage:
-----------

# Query by object name
result = ZTFBTS.query_object('ZTF18aaqedfj')

# Query by position
from astropy.coordinates import SkyCoord
import astropy.units as u
coord = SkyCoord(ra=202.469, dec=47.195, unit=(u.deg, u.deg))
result = ZTFBTS.query_region(coord, radius=5*u.arcmin)

# Query by classification
result = ZTFBTS.query_by_type('SN Ia')

# Query by redshift
result = ZTFBTS.query_by_redshift(z_min=0.05, z_max=0.1)

# Complex multi-criteria query
result = ZTFBTS.query_criteria(
    classification='SN Ia',
    z_min=0.05,
    z_max=0.1,
    peak_mag_max=18.0,
    rise_min=5.0,
    rise_max=20.0
)

# Get light curve
lc = ZTFBTS.get_light_curve('ZTF18aaqedfj')

NOTES
=====

API Implementation:
------------------
The module implements a proposed JSON API structure based on the ZTF BTS
Explorer's query parameters. The actual Explorer currently uses a web form
interface. To connect to the real service, an API endpoint would need to be
implemented by the ZTF BTS team, or the module could be adapted to parse
the web interface.

All query payload generation is fully functional and can be inspected using:
    payload = ZTFBTS.query_object('ZTF18aaqedfj', get_query_payload=True)
